# main分支版本回溯
## 4.13经过feature/navigation的合并
在main分支
```bash
git log main --oneline -20
```
会输出一系列commit记录，每行前面是指针，后面是commit信息
最后一个字母是`:`
这是less分页器，:后面是等待你输入的指令
| 指令 | 描述 |
| --- | --- |
| q | 退出less分页器 |
| 空格 | 下一页 |
| b | 上一页 |
| 上键 | 上一行 |
| 下键 | 下一行 |
| /关键字 | 搜索关键字 |

- HEAD指针指向当前所在版本
    - 如果HEAD指针正常，他应该指向某个分支的最新commit
    - 分离HEAD状态：直接指向某个commit，没有指向任何分支
       - ==**此时代码只能读取和编译，不能做任何修改**==
       - ==**如果需要修改，需要新建一个分支，比如`main-4.13-fixbug`，然后在新分支上做修改，再合并到main分支**==
    - 查看 HEAD 位置:`git log --oneline`

- ==版本回溯==：
    - 找到你要回溯的commit，记住指针
    - 输入`git checkout 指针`
    - 切换回最新commit：`git checkout main`
    - 切换到上一个切换的分支：`git checkout -`
    - 查看HEAD在哪里：`git log --oneline`
       - 正常状态：`18b21da (HEAD -> main, origin/main, origin/HEAD)`
       - 分离HEAD状态：`4b3e608 (HEAD)`
    - 相对切换：
```bash
git checkout HEAD~1     # 上一个 commit
git checkout HEAD~3     # 往前第 3 个 commit
git checkout HEAD~10    # 往前第 10 个 commit   
```
